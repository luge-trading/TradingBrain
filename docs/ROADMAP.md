# TradingBrain 路线图：TASK-012 市场与板块环境分析

## 1. 目标与实施原则

TASK-012 的目标是为现有个股复盘增加可解释的市场和板块环境，使报告能够说明指数、成交活跃度、市场宽度、板块强弱及个股相对表现。它不改变项目“不自动下单、不保证收益、不输出确定性买卖结论”的边界。

TASK-012 拆分为 TASK-012A 至 TASK-012E。每次只实施一个子任务，并遵循：

1. 先形成任务设计、数据口径、schema、异常策略和验收标准；
2. 设计确认后再编码，不跨任务提前实现；
3. 单元测试不访问真实网络，使用 mock、固定样本和临时 SQLite；
4. 真实数据验证单独授权、限次执行，并与自动化测试分离；
5. 每个子任务独立测试、受控暂存、commit 和普通 push；
6. 单一指数、市场字段、板块或个股失败不能破坏其他已成功结果；
7. 所有规则必须可解释，不引入黑箱预测。

## 2. 全局数据口径

- 交易日基准沿用 XSHG 日历；跨市场数据必须明确自身交易日差异和对齐方式。
- 业务日期统一使用 `trade_date`，存储为可排序的 ISO 日期；运行时间使用带时区时间。
- 指数代码、市场代码、板块标识和个股代码必须使用稳定、可校验的内部表示。
- 原始行情、来源元数据和派生指标分层，避免把计算结果伪装为数据源原始字段。
- 涨跌幅、成交额和上涨/下跌家数必须记录数据源定义与统计范围；未能确认完整口径的指标不得进入核心 schema。
- 缺失、休市、接口延迟、字段变更和采集失败必须与合法的零值区分。
- 不把单一“资金流入”字段作为市场或板块强弱的确定性依据。

具体字段和阈值必须在各子任务设计阶段依据受控样本确认，不能仅凭接口名称推断。

## 3. 数据源选择边界

优先评估复用现有 EastMoney 接入方式，以降低依赖数量和运行复杂度，但每类数据仍须独立验证：

- 接口稳定性、字段含义和代码映射；
- 前复权、不复权或指数原始点位等价格口径；
- 成交额单位、统计范围和更新时间；
- 历史数据长度、分页和限流行为；
- 休市日、停牌、空响应和字段缺失行为。

若 EastMoney 无法提供稳定且口径清晰的数据，可在对应子任务设计中比较其他公开数据源。不得静默混用不同来源，也不得在本路线图中提前承诺未经验证的具体接口。

生产数据源访问必须具有超时、有限重试、安全错误和明确 source 标识。禁止无限重试或以陈旧数据伪装本次更新成功。

## 4. SQLite 设计原则

TASK-012 不直接扩展现有 `stock_daily` 字段。计划按数据域建立独立表，具体 schema 在对应子任务确认：

- 指数日线：稳定指数标识与 `trade_date` 组成业务唯一键；
- 市场宽度：市场范围与 `trade_date` 组成业务唯一键；
- 板块日线：板块类型、稳定板块标识与 `trade_date` 组成业务唯一键；
- 个股板块归属：个股、板块类型、板块标识和有效时间信息；
- 派生结果：优先运行时计算；只有明确复用和审计需求时才持久化。

所有表应支持：

- 幂等 upsert 和增量更新；
- 参数化 SQL、事务安全和稳定排序；
- source 及必要的采集元数据；
- 临时数据库测试和 schema 回归；
- 与现有 `stock_daily` 和报告读取逻辑兼容。

不得在未设计迁移与回归测试时修改既有生产 schema。

## 5. 异常与降级策略

- 单一对象失败：记录对象、阶段和安全错误，继续处理其他对象。
- 批次部分失败：返回结构化成功/失败集合，并在报告中标记“部分失败”。
- 数据源整体失败：不覆盖已有有效数据，不生成虚假的本日成功状态。
- 缺失数据：显式显示“不可用”及原因，不用零值或上一日值静默填充。
- 字段或口径变化：校验失败并停止受影响的数据写入，保留诊断信息。
- 报告与邮件：新增市场数据失败不得破坏已有个股报告；邮件失败继续沿用 TASK-011D 隔离语义。

## 6. 测试与真实验证边界

每个子任务至少覆盖：

- 响应解析、字段校验、单位和代码映射；
- 空响应、缺失字段、异常值和重复记录；
- SQLite 初始化、幂等保存、增量读取和事务失败；
- 单对象失败与批次部分失败；
- CLI 或公共接口的参数和退出码；
- 与现有全量测试的回归兼容。

单元测试不得依赖真实网络、真实用户数据库、真实 LaunchAgent、Keychain 或邮件通道。真实数据探针必须在自动化测试全部通过后受控执行，限制对象、日期范围、超时、重试和次数，并且不得污染 Git 状态。

## 7. TASK-012A：核心指数数据（已完成、提交并推送）

### 目标

建立核心指数日线的获取、标准化、SQLite 存储、增量更新、读取和基础指标能力。

首批范围固定为：

- 上证指数；
- 深证成指；
- 创业板指；
- 科创 50。

沪深 300、中证 500、中证 1000 仅作为后续可扩展候选，本阶段不承诺实现。

### 依赖

- 复用现有数据提供者、SQLite 和增量更新的设计经验；
- 沿用 XSHG 日期与 Asia/Shanghai 时间约定；
- 不修改 `stock_daily` 语义。

### 输入

- 经确认的内部指数代码与数据源代码映射；
- 日期范围或最新已存交易日；
- 数据源返回的指数 OHLC、成交量/成交额及日期字段。

### 输出

- 标准化指数日线结构；
- 独立指数日线 SQLite 表；
- 保存、读取、最新日期和增量更新接口；
- 可复用现有技术指标定义的基础指数指标，但需先验证适用性；
- 结构化更新结果和安全错误。

### 已完成验收

- 四个首批指数具有稳定映射和字段口径文档；
- schema 独立、唯一键明确、写入幂等；
- 增量更新不会重复或覆盖无关指数；
- 单一指数失败不阻断其他指数；
- 空数据和字段变化不会写入虚假成功记录；
- 四指数固定注册表、严格日K校验、独立 SQLite 表和幂等 upsert 已完成；
- 历史修订可覆盖，且 `new_rows` 只统计新增交易日；
- 无网络单元测试覆盖解析、存储、更新和失败路径；
- 四指数受控真实 provider 探针已完成；
- 临时 SQLite 往返和指标包装通过；
- 不提前实现市场宽度、板块分析或报告集成。

收盘前请求可能返回当日尚未最终形成的日K；TASK-012A 接口不做收盘判断，正式最终性由后续调度门控或显式盘中预览标记保证。

正式 commit：`aeb67307f7f2d25d2e0397a789d6ded77f7238c7`（`add core index data infrastructure`）。

## 8. TASK-012B：市场基础环境数据层（已完成、提交并推送）

### 目标

建立可解释的市场基础事实，包括沪市成交额、深市成交额、两市合计成交额，以及收盘后沪深上涨、下跌和平盘家数快照。本任务不生成市场强弱结论，不接入报告或自动调度。

成交额来自 SSE、SZSE 官方股票日度概况，官方亿元值以 `Decimal` 定点换算为整数人民币元。不得把指数 `volume` 或 `amount` 当作两市总成交额。深市范围已确认包含创业板和存托凭证，B 股是否包含尚未确认，不作无证据推断。

市场广度来自 EastMoney `1.000001` 和 `0.399001` 两条记录，对各自 `f104`、`f105`、`f106` 独立校验后求和。接口没有可靠业务日期；`trade_date` 由通过交易日和收盘门控的系统运行上下文赋予，表示采集归属日期而非接口返回日期。TASK-012B 本身不接入该门控。

### 依赖

- TASK-012A 的日期、SQLite 增量初始化、幂等 upsert 和失败保护约定；
- SSE、SZSE 官方日度股票概况及 EastMoney 市场广度快照。

### 输入

- 由调用方明确提供的 ISO `trade_date`；
- SSE 响应中的 `TRADE_DATE`、`TRADE_AMT`；
- SZSE `conditions[name=txtQueryDate].defaultValue` 及“成交金额（亿元）”行的 `gp`；
- EastMoney 沪深两条快照中的 `f104`、`f105`、`f106`。

### 输出

- 独立 `market_daily` 表，以 `trade_date` 为主键；
- `sh_amount_yuan`、`sz_amount_yuan`、`total_amount_yuan`；
- `advance_count`、`decline_count`、`flat_count`；
- 拆分的 SSE、SZSE、EastMoney source 字段及 `updated_at`；
- 读取时派生的 nullable `advance_ratio`；
- 独立 `update_market_daily()` 结构化尝试值、存储值和错误结果。

### 验收标准

- SSE、SZSE 日期分别按官方响应字段和 SZSE 查询条件元数据严格匹配请求日期；
- 亿元到人民币元禁止 float 累乘，保留官方两位小数精度；
- 两市金额齐全才计算合计，广度三字段整体成功或整体缺失；
- 合法零值与 SQLite `NULL` 可区分，旧有效数据不会被失败组覆盖；
- `advance_ratio` 不持久化，任一计数字段缺失或分母不大于零时返回缺失；
- SQLite 初始化只新增表，不修改或破坏 `stock_daily`、`index_daily`；
- provider 重试、非法响应、事务回滚和部分失败均由无网络测试覆盖；
- 全量离线验收为 `307 passed`；SSE、SZSE 受控真实探针通过；
- EastMoney 真实探针因当前 Python `requests` 代理链路 `RemoteDisconnected` 被阻塞，按环境限制记录验收豁免，不据此放宽实现校验。

### 明确延期范围

TASK-012B 不实现 `limit_up_count`、`limit_down_count` 或证券涨跌停规则计算。当前没有经验证的数据源同时满足完整沪深市场覆盖、明确计算定义和稳定历史查询；EastMoney 非 ST 涨跌停专题池不作为替代。完整涨跌停统计如需实现，应作为独立任务重新设计，不默认纳入 TASK-012C。

正式 commit：`c1fec1d2c8b8ea5237e04ed8ea3898a7ded264cd`（`add market environment data layer`）。

## 9. TASK-012C：东方财富三级行业注册表与历史日线事实层（编码与离线验收完成，真实探针环境豁免）

### 目标

建立东方财富一级、二级、三级行业的当前注册表和不复权历史日线事实层。本任务只保存可验证事实，不形成行业排名、相对强弱、热点或主线判断。

### 依赖

- TASK-012A 的行情标准化、SQLite 幂等 upsert 和历史修订语义；
- 已受控确认的 EastMoney 三级行业过滤条件、分页规则及 K 线身份字段。

### 输入

- 分层行情注册表中的行业代码和当前名称；
- `90.<BK代码>` 对应的不复权日 K 响应。

### 输出

- `sector_registry` 当前注册表、来源与 active/inactive 状态；
- `sector_daily` 不复权 OHLC、nullable provider-native volume/amount 和原始 change_pct；
- 单行业和串行批量更新的结构化成功/失败结果。

### 验收标准

- 一级、二级、三级分别独立分页，不使用混合列表或名称后缀猜测层级；
- 代码、响应身份、日期、OHLC 和 nullable 数值严格校验；
- 完整注册表快照原子保存，退出或迁级代码标为 inactive 而不删除历史；
- 行业日 K 幂等 upsert 并允许历史修订，单行业失败不破坏其他行业；
- 无网络定向测试为 `148 passed`，全量离线测试为 `412 passed`；
- 2026 年 7 月 21 日受控真实探针在首个一级注册表请求取得 HTTP/JSON 响应前被当前 Python `requests` 代理链路阻塞；该结果不证明正式 provider 正确或错误，已按环境限制记录验收豁免。

本任务不实现概念、父子关系、行业成分股、个股映射、排名、相对收益、强弱判断、资金流或报告集成。注册表名称只代表当前名称，不提供历史名称语义。

## 10. TASK-012D：行业相对强弱与排名

### 目标

基于 TASK-012C 的行业历史事实和已完成的指数、市场环境事实，设计透明、可复现的行业相对表现和排名。本阶段仍不进行个股映射或热点、主线、龙头判断。

### 依赖

- TASK-012C 的 active 行业注册表和历史日线；
- TASK-012A、TASK-012B 的市场背景事实及对齐日期。

### 输入

- 同层级行业的可比日期窗口；
- 经设计确认的收益、持续性和数据完整性口径。

### 输出

- 分层行业相对表现和排名；
- 排名所用事实、窗口、缺失原因和可审计解释字段。

### 验收标准

- 不同层级不混合排名；
- 历史窗口、缺失和并列规则明确；
- 算法透明、固定输入可复现并具有边界测试；
- 输出是事实派生研究描述，不是确定性交易信号。

## 11. TASK-012E：全市场个股横截面

### 目标

建立全市场个股横截面事实基础，为后续个股与行业关系、相对表现和研究筛选提供可追溯输入。本任务的精确股票池、字段、复权和交易状态口径须另行设计冻结。

### 依赖

- TASK-012A 至 TASK-012D 的稳定事实和行业排名接口；
- 经验证的全市场股票池与日度事实数据源。

### 输入

- 可追溯的全市场股票列表；
- 日期一致、口径明确的个股横截面事实。

### 输出

- 结构化全市场个股横截面；
- 明确的数据缺失、停牌和异常状态；
- 供后续个股—行业联动使用的稳定读取接口。

### 验收标准

- 股票池范围、日期、价格和交易状态口径均有证据；
- 不用零值或旧值伪装缺失事实；
- 批次失败隔离且不会污染已有数据；
- 单元测试无网络，真实验证限次且不写正式数据库。

## 12. 依赖关系与阶段门

主依赖顺序：

```text
TASK-012A 核心指数数据
  → TASK-012B 市场基础环境数据层
  → TASK-012C 东方财富三级行业事实层
  → TASK-012D 行业相对强弱与排名
  → TASK-012E 全市场个股横截面
```

每个阶段进入编码前必须单独确认：任务边界、数据源、字段口径、SQLite schema、异常策略、测试方案和受控真实验证计划。当前必须先完成 TASK-012C 的受控暂存和提交闭环；闭环后的下一个唯一任务是 TASK-012D，仍须先设计、后编码。任何后续正式报告或自动化集成都必须继承交易日/收盘门控要求，但不在上述三个事实/分析任务中提前实现。
