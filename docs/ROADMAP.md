# TradingBrain 路线图：TASK-012 市场与板块环境分析

## 1. 目标与实施原则

TASK-012 的目标是为现有个股复盘增加可解释的市场和板块环境，使报告能够说明指数、成交活跃度、市场宽度、板块强弱及个股相对表现。它不改变项目“不自动下单、不保证收益、不输出确定性买卖结论”的边界。

TASK-012 拆分为 TASK-012A 至 TASK-012E。每次只实施一个子任务，并遵循：

1. 先形成任务设计、数据口径、schema、异常策略和验收标准；
2. 设计确认后再编码，不跨任务提前实现；
3. 单元测试不访问真实网络，使用 mock、固定样本和临时 SQLite；
4. 真实数据验证单独授权、限次执行，并与自动化测试分离；
5. 每个子任务独立测试、受控暂存、commit 和普通 push；
6. 单一指数、市场字段、板块或个股失败不能破坏其他已成功结果；
7. 所有规则必须可解释，不引入黑箱预测。

## 2. 全局数据口径

- 交易日基准沿用 XSHG 日历；跨市场数据必须明确自身交易日差异和对齐方式。
- 业务日期统一使用 `trade_date`，存储为可排序的 ISO 日期；运行时间使用带时区时间。
- 指数代码、市场代码、板块标识和个股代码必须使用稳定、可校验的内部表示。
- 原始行情、来源元数据和派生指标分层，避免把计算结果伪装为数据源原始字段。
- 涨跌幅、成交额、上涨/下跌家数和涨跌停数量必须记录数据源定义与统计范围。
- 缺失、休市、接口延迟、字段变更和采集失败必须与合法的零值区分。
- 不把单一“资金流入”字段作为市场或板块强弱的确定性依据。

具体字段和阈值必须在各子任务设计阶段依据受控样本确认，不能仅凭接口名称推断。

## 3. 数据源选择边界

优先评估复用现有 EastMoney 接入方式，以降低依赖数量和运行复杂度，但每类数据仍须独立验证：

- 接口稳定性、字段含义和代码映射；
- 前复权、不复权或指数原始点位等价格口径；
- 成交额单位、统计范围和更新时间；
- 历史数据长度、分页和限流行为；
- 休市日、停牌、空响应和字段缺失行为。

若 EastMoney 无法提供稳定且口径清晰的数据，可在对应子任务设计中比较其他公开数据源。不得静默混用不同来源，也不得在本路线图中提前承诺未经验证的具体接口。

生产数据源访问必须具有超时、有限重试、安全错误和明确 source 标识。禁止无限重试或以陈旧数据伪装本次更新成功。

## 4. SQLite 设计原则

TASK-012 不直接扩展现有 `stock_daily` 字段。计划按数据域建立独立表，具体 schema 在对应子任务确认：

- 指数日线：稳定指数标识与 `trade_date` 组成业务唯一键；
- 市场宽度：市场范围与 `trade_date` 组成业务唯一键；
- 板块日线：板块类型、稳定板块标识与 `trade_date` 组成业务唯一键；
- 个股板块归属：个股、板块类型、板块标识和有效时间信息；
- 派生结果：优先运行时计算；只有明确复用和审计需求时才持久化。

所有表应支持：

- 幂等 upsert 和增量更新；
- 参数化 SQL、事务安全和稳定排序；
- source 及必要的采集元数据；
- 临时数据库测试和 schema 回归；
- 与现有 `stock_daily` 和报告读取逻辑兼容。

不得在未设计迁移与回归测试时修改既有生产 schema。

## 5. 异常与降级策略

- 单一对象失败：记录对象、阶段和安全错误，继续处理其他对象。
- 批次部分失败：返回结构化成功/失败集合，并在报告中标记“部分失败”。
- 数据源整体失败：不覆盖已有有效数据，不生成虚假的本日成功状态。
- 缺失数据：显式显示“不可用”及原因，不用零值或上一日值静默填充。
- 字段或口径变化：校验失败并停止受影响的数据写入，保留诊断信息。
- 报告与邮件：新增市场数据失败不得破坏已有个股报告；邮件失败继续沿用 TASK-011D 隔离语义。

## 6. 测试与真实验证边界

每个子任务至少覆盖：

- 响应解析、字段校验、单位和代码映射；
- 空响应、缺失字段、异常值和重复记录；
- SQLite 初始化、幂等保存、增量读取和事务失败；
- 单对象失败与批次部分失败；
- CLI 或公共接口的参数和退出码；
- 与现有全量测试的回归兼容。

单元测试不得依赖真实网络、真实用户数据库、真实 LaunchAgent、Keychain 或邮件通道。真实数据探针必须在自动化测试全部通过后受控执行，限制对象、日期范围、超时、重试和次数，并且不得污染 Git 状态。

## 7. TASK-012A：核心指数数据（实现与验证完成，待本任务受控提交）

### 目标

建立核心指数日线的获取、标准化、SQLite 存储、增量更新、读取和基础指标能力。

首批范围固定为：

- 上证指数；
- 深证成指；
- 创业板指；
- 科创 50。

沪深 300、中证 500、中证 1000 仅作为后续可扩展候选，本阶段不承诺实现。

### 依赖

- 复用现有数据提供者、SQLite 和增量更新的设计经验；
- 沿用 XSHG 日期与 Asia/Shanghai 时间约定；
- 不修改 `stock_daily` 语义。

### 输入

- 经确认的内部指数代码与数据源代码映射；
- 日期范围或最新已存交易日；
- 数据源返回的指数 OHLC、成交量/成交额及日期字段。

### 输出

- 标准化指数日线结构；
- 独立指数日线 SQLite 表；
- 保存、读取、最新日期和增量更新接口；
- 可复用现有技术指标定义的基础指数指标，但需先验证适用性；
- 结构化更新结果和安全错误。

### 已完成验收

- 四个首批指数具有稳定映射和字段口径文档；
- schema 独立、唯一键明确、写入幂等；
- 增量更新不会重复或覆盖无关指数；
- 单一指数失败不阻断其他指数；
- 空数据和字段变化不会写入虚假成功记录；
- 四指数固定注册表、严格日K校验、独立 SQLite 表和幂等 upsert 已完成；
- 历史修订可覆盖，且 `new_rows` 只统计新增交易日；
- 无网络单元测试覆盖解析、存储、更新和失败路径；
- 四指数受控真实 provider 探针已完成；
- 临时 SQLite 往返和指标包装通过；
- 不提前实现市场宽度、板块分析或报告集成。

收盘前请求可能返回当日尚未最终形成的日K；TASK-012A 接口不做收盘判断，正式最终性由后续调度门控或显式盘中预览标记保证。

## 8. TASK-012B：两市成交额与市场宽度

### 目标

建立可解释的两市成交活跃度与市场宽度数据，包括两市成交额、上涨家数、下跌家数、涨停家数和跌停家数，并基于透明规则形成市场强弱与风险状态。

设计前置条件：必须先明确数据来源、统计日期、单位和统计口径。不得把指数 `volume` 或 `amount` 直接当作两市总成交额；若数据源缺少权威单位或盘后最终性证明，必须显式标记缺失/不确定，不得推断。

### 依赖

- TASK-012A 的交易日期、更新编排和市场环境数据存储约定。

### 输入

- 数据源明确统计范围的沪深市场每日汇总；
- 成交额单位、股票样本范围、涨跌停定义和交易状态字段。

### 输出

- 独立的每日市场宽度记录；
- 成交额与宽度派生指标；
- 可解释的市场强弱/风险状态及触发原因。

### 验收标准

- 各统计字段口径、单位和覆盖范围有文档；
- 合法零值与缺失/失败可区分；
- 规则阈值透明且有边界测试；
- 数据源失败不影响已有指数和个股数据；
- 无网络测试与受控真实验证完整。

## 9. TASK-012C：行业与概念板块强弱

### 目标

获取行业和概念板块的日度表现，形成基于涨跌、成交活跃度和横向排名的板块强弱视图。

### 依赖

- TASK-012A 的日期和行情规范；
- TASK-012B 的市场环境口径，用于解释板块表现所处背景。

### 输入

- 行业和概念板块的稳定标识、名称及日度行情；
- 涨跌幅、成交额和经过验证的可解释字段。

### 输出

- 行业/概念板块日线；
- 板块排名、强弱等级及解释字段；
- 部分失败和板块更名/映射异常记录。

### 验收标准

- 板块类型与标识稳定，不只依赖展示名称；
- 行业与概念口径不混用；
- 排名结果在固定输入下可复现；
- 不以单一资金流字段得出确定性结论；
- 单一板块失败不破坏批次；
- 无网络测试和受控真实验证覆盖核心路径。

## 10. TASK-012D：个股与板块联动

### 目标

将个股所属行业和概念与板块强弱连接，识别个股相对板块强弱、板块上涨但个股走弱等背离，并说明个股是否处于当前强势板块集合。

### 依赖

- TASK-012C 的板块标识、板块日线和排名；
- 现有个股信号与报告结构。

### 输入

- 个股到行业/概念板块的可追溯映射；
- 个股和对应板块同日表现；
- 板块排名与市场背景。

### 输出

- 个股相对板块表现；
- 背离与同向强弱等解释性标签；
- 多板块归属的结构化结果和来源说明。

### 验收标准

- 映射来源、更新时间和多板块规则明确；
- 缺失映射不会伪造归属；
- 相对强弱算法透明、可复现并有边界测试；
- 输出是研究描述，不是确定性买卖信号；
- 单只股票或板块失败保持隔离。

## 11. TASK-012E：市场环境汇总报告

### 目标

将指数、成交额、市场宽度、板块强弱和个股板块联动整合为每日市场环境报告，并接入现有每日汇总与邮件摘要。

### 依赖

- TASK-012A 至 TASK-012D 全部完成并具有稳定结构化输出；
- 现有 daily summary、scheduled review 和邮件失败隔离语义。

### 输入

- 当日指数与基础指标；
- 两市成交额和市场宽度；
- 行业/概念板块排名；
- 自选股与板块联动结果。

### 输出

- 每日市场环境 Markdown 报告；
- 现有每日汇总中的市场环境章节；
- 邮件正文中的精简市场环境摘要；
- 数据缺失和部分失败说明。

### 验收标准

- 报告内容完全来自结构化结果，不重新解析 Markdown；
- 部分市场数据失败时仍可生成带明确缺失说明的报告；
- 不改变现有个股复盘退出码和邮件失败隔离；
- skipped 状态不产生无意义邮件；
- 旧报告和现有 CLI 保持兼容；
- 单元测试无网络，真实整链验证受控且不得重复发送测试邮件。

## 12. 依赖关系与阶段门

主依赖顺序：

```text
TASK-012A 核心指数数据
  → TASK-012B 两市成交额与市场宽度
  → TASK-012C 行业与概念板块强弱
  → TASK-012D 个股与板块联动
  → TASK-012E 市场环境汇总报告
```

每个阶段进入编码前必须单独确认：任务边界、数据源、字段口径、SQLite schema、异常策略、测试方案和受控真实验证计划。当前下一个唯一任务是 TASK-012B；TASK-012E 必须继承交易日/收盘门控要求。未经确认，不直接开始 TASK-012B 编码。
